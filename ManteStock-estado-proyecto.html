Decisiones de arquitectura clave (registradas con fecha)

  • Roles: TECNICO, COORDINADOR, ADMIN. Solo el CRUD de usuarios (/users) está protegido por rol; Materiales e Inventory Movements siguen abiertos a cualquier usuario autenticado (decisión consciente, no un olvido).
  • Trazabilidad (createdBy): se guarda como texto plano (username), no como relación a User — evita joins y el riesgo de exponer el hash de contraseña.
  • Sesión del frontend: cookie simple (no httpOnly) en vez de localStorage, necesario para que el middleware de rutas protegidas y los Server Components puedan leerla. Trade-off de seguridad aceptado y documentado para el alcance de MVP.
  • Inmutabilidad de movimientos: confirmado por especificación (RNF08) — no se pueden editar ni eliminar, por diseño, no por limitación técnica.
  • Stock como entero: se difiere el soporte de decimales (kg, litros fraccionados) — el catálogo asume unidades enteras por ahora.
