ManteStock — Guía funcional para el rol Administrador

Todo lo que puede hacer un Técnico o Coordinador, más la gestión de usuarios del sistema

El Administrador tiene el mismo acceso funcional que Técnico y Coordinador sobre Materiales e Inventario (no hay restricciones adicionales de rol en esos módulos). Esta guía se enfoca en lo que es exclusivo del rol Administrador; para el resto (materiales, movimientos, dashboard), la guía del Técnico aplica igual.

1. Iniciar sesión

1
Ingresa con tu usuario y contraseña de administrador. Igual que cualquier rol, dos intentos fallidos seguidos bloquean la cuenta por 5 minutos.
2
En el encabezado, arriba a la derecha, verás tu usuario (ADMIN) y un enlace adicional Usuarios en el menú — ese enlace solo lo ve el rol Administrador.

2. Ver la lista de usuarios

1
Entra a Usuarios en el menú superior.
2
Verás una tabla de solo lectura con: nombre de usuario, rol, y si está activo o no.
Esta pantalla es solo de consulta. Crear usuarios, cambiar su rol, resetear contraseñas o desactivarlos todavía no tiene botones en la interfaz — esas acciones se hacen por API (ver sección 3). Si un Técnico o Coordinador intenta entrar a /users directamente por la URL, el sistema lo redirige y no le muestra nada.

3. Gestionar usuarios (por API, con Postman u otra herramienta similar)

Mientras no exista la pantalla de gestión en el frontend, estas acciones se hacen enviando peticiones directas a http://localhost:3001, incluyendo el token de tu sesión de administrador en el header Authorization: Bearer <token>.

Crear un usuario nuevo

POST /users
{
  "username": "tecnico2",
  "password": "Clave123",
  "role": "TECNICO"
}
CampoRegla
usernameDebe ser único — si ya existe, el sistema lo rechaza.
passwordMínimo 8 caracteres, con al menos una mayúscula, una minúscula y un número.
roleTECNICO, COORDINADOR o ADMIN.

Cambiar el rol o desactivar un usuario

PATCH /users/:id
{ "role": "COORDINADOR" }

PATCH /users/:id
{ "isActive": false }
Un administrador no puede desactivarse a sí mismo (ni con este PATCH ni con el DELETE de abajo) — el sistema lo rechaza, para que nunca quede el sistema sin ningún administrador activo.

Dar de baja a un usuario

DELETE /users/:id

No borra el registro — lo marca como inactivo (isActive: false). El usuario deja de poder iniciar sesión, pero su historial de movimientos (createdBy) se conserva intacto.

Resetear la contraseña de un usuario

PATCH /users/:id/reset-password
{ "newPassword": "NuevaClave123" }

Es el único mecanismo de recuperación de contraseña que existe hoy — no hay recuperación por correo electrónico. Si un usuario olvida su clave, el administrador se la resetea manualmente por acá.

4. Qué NO cambia para el Administrador

  • Los movimientos de inventario siguen siendo inmutables — ni siquiera el Administrador puede editarlos o eliminarlos una vez guardados.
  • El stock de un material solo cambia registrando un movimiento (entrada/salida/ajuste), nunca editando el material directamente.

5. Cerrar sesión

Presiona Cerrar sesión arriba a la derecha en cualquier pantalla.

Guía basada en el comportamiento real verificado del sistema (backend con Postman, frontend en navegador) — no en la especificación original. La gestión de usuarios por API se documenta aquí porque, a la fecha, no existe interfaz gráfica para esas acciones.
