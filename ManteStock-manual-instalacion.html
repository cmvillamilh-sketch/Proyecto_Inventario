ManteStock — Manual de instalación desde cero

Pasos para levantar el proyecto completo (backend + frontend + base de datos) en un computador nuevo

El código todavía no está mezclado en main. Hoy vive en la rama feature/checkpoints-007-010, pendiente de Pull Request. Este manual asume que se instala desde esa rama.

1. Requisitos previos

HerramientaVersión / detalle
Node.js20.x — obligatorio, versiones distintas dan problemas de compatibilidad con Next.js 14 en Windows.
npmViene incluido con Node.js.
Docker DesktopPara levantar PostgreSQL en contenedor (opción recomendada). Alternativa: PostgreSQL 16 instalado localmente.
GitPara clonar el repositorio.
1
Verifica tu versión de Node: 
node -v
Debe mostrar v20.x.x. Si tienes otra versión, instala Node 20 (por ejemplo con nodejs.org o un gestor de versiones como nvm).

2. Clonar el repositorio

1
Clona el repositorio y entra a la carpeta: 
git clone https://github.com/cmvillamilh-sketch/Proyecto_Inventario.git
cd Proyecto_Inventario
2
Cambia a la rama de trabajo (el checkpoint más reciente no está en main todavía): 
git checkout feature/checkpoints-007-010

3. Configurar variables de entorno

1
Copia el archivo de ejemplo a .env en la raíz del proyecto: 
copy .env.example .env
(en Mac/Linux sería cp .env.example .env)
2
Revisa el contenido de .env: 
POSTGRES_DB=mante_stock
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_PORT=5432
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/mante_stock
JWT_SECRET=change-me
NEXT_PUBLIC_API_URL=http://localhost:3001
Para levantar el proyecto en tu propio equipo, estos valores por defecto sirven tal cual. Solo cámbialos si ya tienes algo corriendo en esos puertos.
JWT_SECRET=change-me es un valor de desarrollo, aceptado para instalar y probar el proyecto localmente. Si este proyecto llegara a desplegarse en un servidor real, ese valor debe reemplazarse por un secreto fuerte y único.

4. Instalar dependencias

1
Desde la raíz del proyecto (es un monorepo con npm Workspaces, un solo install cubre backend y frontend): 
npm install

5. Levantar la base de datos

1
Con Docker Desktop abierto, levanta PostgreSQL 16 en contenedor: 
npm run docker:up
Esto crea el contenedor mante-stock-postgres, escuchando en el puerto 5432.
No hace falta crear tablas a mano: el backend usa synchronize: true en desarrollo, así que TypeORM crea el esquema automáticamente la primera vez que arranca.

6. Crear usuarios (administrador, técnico, coordinador)

No existe una pantalla de registro pública (a propósito — la creación de usuarios es admin-only). Hay dos formas de crear usuarios; usa la que corresponda en cada caso.

6.1 Primer usuario (obligatorio) — script de bootstrap

Antes de tener un administrador, no hay forma de autenticarse contra la API para crear usuarios por ese camino. Por eso el primer usuario (debe ser ADMIN) se crea con un script de bootstrap, ejecutado directo contra la base de datos:

1
Con la base de datos ya levantada (paso 5), ejecuta: 
npm run seed:test-user --workspace apps/backend -- admin Admin123! ADMIN
Los tres argumentos son opcionales — si no los pasas, usa por defecto admin / Admin123! / ADMIN.
2
La consola confirma el usuario creado (o avisa si ese username ya existía, sin duplicarlo). Guarda esas credenciales.

6.2 Resto de usuarios (técnico, coordinador, u otros admin) — dos opciones

El script de bootstrap no está limitado a un solo uso: acepta cualquier username/password/role, así que también sirve para crear técnicos y coordinadores de prueba sin necesidad de Postman. La vía por API es la que realmente usará un administrador ya en funcionamiento, porque queda autenticada y registrada como acción suya.

Opción A — repetir el script de bootstrap (rápido, para poblar datos de prueba):

npm run seed:test-user --workspace apps/backend -- tecnico1 Tecnico123 TECNICO
npm run seed:test-user --workspace apps/backend -- coordinador1 Coord123X COORDINADOR

Opción B — API, con el backend corriendo (paso 7), usando el token del administrador creado en 6.1:

1
Inicia sesión como administrador (POST /auth/login o desde /login en el navegador) y copia el accessToken que devuelve.
2
Con Postman (u otra herramienta), envía, incluyendo el header Authorization: Bearer <accessToken>: 
POST http://localhost:3001/users
{
  "username": "tecnico1",
  "password": "Tecnico123",
  "role": "TECNICO"
}
role acepta TECNICO, COORDINADOR o ADMIN. La contraseña debe tener mínimo 8 caracteres, con mayúscula, minúscula y número.
3
Repite el paso anterior por cada usuario que necesites. El detalle completo de gestión de usuarios (cambiar rol, desactivar, resetear contraseña) está en la guía funcional del Administrador.

7. Arrancar backend y frontend

Se ejecutan en dos terminales separadas, ambas desde la raíz del proyecto:

TerminalComandoPuerto
Backend (NestJS)npm run dev:backendhttp://localhost:3001
Frontend (Next.js)npm run dev:frontendhttp://localhost:3000
El backend NO responde en el puerto 3000 — ese puerto es del frontend. Si vas a probar la API con Postman o similar, usa siempre http://localhost:3001.

8. Verificar que quedó funcionando

1
Backend: abre http://localhost:3001/health en el navegador. Debe responder algo como: 
{ "status": "ok", "service": "mante-stock-backend", "timestamp": "..." }
2
Frontend: abre http://localhost:3000. Debe redirigirte a la pantalla de login.
3
Inicia sesión con el usuario administrador creado en el paso 6. Debes llegar al dashboard con las tarjetas de indicadores.

9. Problemas frecuentes

SíntomaCausa probableSolución
El backend no arranca: Error: JWT_SECRET no esta definido Falta el archivo .env o la variable JWT_SECRET Repite el paso 3 — el archivo .env debe estar en la raíz del proyecto.
/materials o /inventory-movements dan error 401 en el frontend No hay sesión iniciada, o el token expiró (dura 8 horas) Vuelve a iniciar sesión desde /login.
Error de conexión a PostgreSQL El contenedor no está corriendo Verifica con docker ps que mante-stock-postgres aparezca activo; si no, repite npm run docker:up.
Errores raros de compatibilidad en Windows con Next.js Versión de Node distinta a 20.x Confirma node -v y reinstala Node 20 si hace falta.
